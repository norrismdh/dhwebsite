How they work

Read-only. Metadata-only. Native auth

Every connector follows the same three rules. No exceptions, no special cases per tool

1. Read-only indexing

We crawl your tool's metadata API on a schedule you set. We never write, never edit, never overwrite. Source platforms can't tell the difference between us and a viewer

2. Metadata only

Titles, owners, descriptions, last-modified, usage signals. Never the data itself. The actual numbers and pixels stay in the source tool - viewed there, edited there

3. Native auth passthrough

When a user opens a report, the source tool authenticates them with their own identity. RLS, OLS, workspace permissions - all inherited automatically. We never escalate privileges
